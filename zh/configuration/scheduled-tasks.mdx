Aether 的计划任务系统让你定义按指定时间自动运行的周期性 Agent 工作流 — 无需手动输入。设置提示词、选择计划，其余交给 Aether，你可专注于其他事情。

## 计划任务如何工作

每个计划任务包含两个核心部分：告诉 Agent 做什么的 **提示词**，以及决定何时执行的 **计划**。到点后，Aether 会将该提示词追加到该任务的持久聊天会话，使用该会话的模型、Skills、MCP 服务器与 Agent 模式状态运行完整的 Agent 循环，并在启用完成通知且 Aether 不在前台时发送通知。

## 创建任务

<Steps>
  <Step title="打开计划任务设置">
    打开 Aether，然后进入 **设置 → 计划任务**。
  </Step>
  <Step title="点击添加任务">
    点击 **添加任务** 按钮，打开任务创建表单。
  </Step>
  <Step title="命名任务并编写提示词">
    输入描述性的 **任务名称**（例如「早间新闻简报」），以及任务触发时希望 Agent 执行的 **提示词**。
  </Step>
  <Step title="设置计划">
    选择任务运行频率：**按间隔**、**每天** 或 **每周**。
  </Step>
  <Step title="使用任务的持久聊天">
    计划任务没有单独的模型或 Skill 选择器。首次运行后，打开该任务的聊天，选择你希望后续运行复用的模型、Skills、MCP 服务器或 Agent 模式状态。
  </Step>
  <Step title="保存任务">
    点击 **保存**。任务现已排程，并将在下一个匹配时间自动触发。
  </Step>
</Steps>

## 查看任务历史

每个计划任务会复用 Aether 对话历史中的一个持久聊天会话。打开 **对话抽屉**（从聊天界面右滑，或点击菜单图标），可在同一对话中查看该任务的全部运行记录。

## 可靠调度的要求

为尽可能可靠地触发计划任务，请检查以下两项 Android 设置：

1. **闹钟调度** — 在提供精确闹钟特殊访问的 Android 版本上，允许该权限可提高定时准确性。若无法使用精确闹钟，Aether 会回退到闹钟时钟或 allow-while-idle 调度。
2. **无电池优化限制** — Android 省电功能可能阻止后台进程准时唤醒。进入 **Android 设置 → 电池 → Aether → 无限制**（措辞可能因厂商而异），允许 Aether 不受限制运行。

---

<Tip>
  计划任务非常适合每日摘要、自动化文件处理，或由 Agent 根据实时数据汇总的早间简报 — 任何你原本每天都要手动启动的事。
</Tip>

<Note>
  即使 Aether 界面在后台，任务仍会运行。强制停止 Aether 或 OEM 激进的后台限制仍可能阻止触发。需要时请将 Aether 加入设备的「受保护应用」或「自启动」列表。
</Note>
