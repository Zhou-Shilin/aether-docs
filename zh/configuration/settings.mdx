Aether 的设置界面是控制应用各方面行为的中心 — 从驱动聊天的 LLM 提供商，到 Agent 的思考方式，再到加载哪些扩展。按工作流定制 Aether 所需的一切都在这里。

## 访问设置

从菜单按钮打开 **对话抽屉**，然后点击 **设置**。设置按清晰分区组织，便于快速找到所需项。

## 设置分区

### 模型提供商

在此添加、编辑与移除 LLM 提供商凭证。每个提供商条目保存连接该服务所需的 API 密钥、OAuth 凭证或环境配置。完整添加与管理流程见 [提供商](/zh/configuration/providers) 页面。

### 默认模型

选择用于新聊天会话、对话标题、自动命名与压缩的默认模型。单个聊天可在顶栏的模型选择器中覆盖聊天默认值，因此全局设置只是起点。

### Agent 模式

通过 Shizuku 或实验性 Root 授权 **Agent 模式**。Agent 模式会添加隔离的虚拟显示工具；Aether 的工具调用循环、shell 命令、文件工具、Skills 与 MCP 服务器无需它即可工作。

### 扩展

浏览 Aether 中当前已安装的全部扩展，从 `.zip` 归档或基于 npm 的扩展目录安装新扩展，并查看加载时记录的扩展错误。你也可在此管理 **Native Mod 安全模式** — 若编译型 Kotlin/DEX mod 导致崩溃，安全模式可在不丢失其他扩展的情况下将其禁用。

### Agent Skills / MCP 服务器

在各自的设置页面管理已安装的 **Agent Skills**，并配置 **MCP（Model Context Protocol）服务器** 连接。Skills 提供可复用的指令与资源。MCP 服务器可按聊天选择。

### 计划任务

设置在指定时间自动运行的周期性 Agent 工作流，无需手动输入。为每个任务配置提示词，以及按间隔、每天或每周的计划。完整说明见 [计划任务](/zh/configuration/scheduled-tasks) 页面。
